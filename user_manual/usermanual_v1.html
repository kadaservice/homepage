üöÄ Overview

This project serves as the official gateway for our ODT application. While the repository is hosted on kadaservice.github.io/homepage, it is configured to securely direct users to our high-performance KADA Cloud.

üõ† How It Works

  1. Frontend Layer: Built and hosted via GitHub Pages for 100% uptime.
  2. Redirection Layer: A script validates the connection and routes the user.
  3. Cloud Layer: The main logic and database reside at: kada cloud system

3. Live Cloud Application

Once the redirect is complete, you will arrive at the main functional dashboard.

3. Modular based DTAM applications

At the left side corner, you can see lists of KADA's products ODT, VDT and CADE. This is for the ODT applcaitons, so you will see available modules in the ODT system.

  1. Operations Features and Setup : You can receive your credentials from the KADA's administration team.
  2. 3D Visulation You can see your specific UAM flight in the 3D visualization module and you can also customize the view according to your preferences. Plus, there you can do the fault injection, weather changes and so on.
  3. Mission Upload Module : You can upload sample mission data for the UAM or you can specify your mission profile data and upload to the specific UAM vehicles.
  4. Situation Awareness Module : This module provides real-time monitoring and alerts for air mobility operations. THis modules have three main parts : (1) Detection and prediction of moving obstacles using Recurrent-YOLO model, (2) Risk assessment (3) advisory recommendation using Adaptive AI model in this module. You can receive images data for AI training or situation data for the futures data analysis from this module.
  5. Report Generation Module : You can received all generated flight data from this module.
